Question: Database Access Pattern

Should we use a Singleton Pattern for the database connection helper to ensure all agents share the same WAL-enabled connection?

A

Singleton Helper (Recommended)

One shared connection with context managers. Safe for multi-agent writes in Zsh/CLI.

B

Per-Call Connections

Open/Close on every query. Simpler but higher overhead and lock risk during heavy sweeps.
